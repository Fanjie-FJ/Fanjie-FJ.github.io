实用操作系统笔记（三）

2019.10.28课前复习

1.段式

进程分段，内存分块
内存块的定义：起始地址+长度
段-》块
虚拟地址-》段号+段内偏移量
物理地址-》无划分（无规则）
段表-》起始地址+长度-〉段对应的内存块
P/M-》存在位和修改位（同页式管理）
地址映射：
虚拟地址-》段号+偏移量-〉段号+段表首地址
（给定：一个进程一个段表）-》段表项-〉首地址+偏移量-》物理地址（P=1）
缺段中断-》缺段处理同缺页处理

2.段页式

进程分段，段内分页，内存分帧，页帧等大小
一个进程：一个段表，多个页表（一个段一个页表）
虚拟地址-》段号+页号+页内偏移量
物理地址-》帧号+帧内偏移量
段表：首地址（指向该段所对应页表的首地址）/长度（该段对应页表的长度）
注意：段表中没有P和M（页表常驻内存）
页表：同页式管理系统（P/M/帧号）
地址映射：
虚拟地址-》段号+段表首地址（给定）-〉段表项-》首地址-〉（页表首地址，页表指针）+页号-》页表项-〉帧号+偏移量-》物理地址（P=1）
缺页中断-》同页式管理（P=0）
在段页式管理系统中，只可能缺页，不可能缺段

3.和内存管理相关的四种策略：

读取策略/放置策略/替换策略/清除策略
（1）读取
研究什么时候从硬盘中把页读到内存中（读：加载，从硬盘拷贝到内存）
（a） 请求式读取：用时才读
缺点：在进程开始运行时，会出现大量的页错误（内存分配给进程时，内存是空的，所有的页都在硬盘上）

​ （b） 预约式读取：提前读
​ 读后有用
​ 读后没用（代价）（没用：后续都没用到或者用到前被替换出去）
​ 减小代价-》前提条件：页在辅存中连续存放（省去寻道时间和旋转延迟）

（2）放置
内存的位置无关性-》随机放置（一般内存不考虑放置算法）
特殊内存：速度与位置有关，离CPU近的快远的慢-》考虑放置算法

（3）替换
帧锁定：帧在内存中不被替换到辅存。（执行频率高/重要的数据结构/管理数据）-》
lock=1 锁定；lock=0 可替换
lock位存放在页表项中
目标：替换掉在最远的将来最不可能被用到的页-》替换出去的页，要在辅存中存放最久再被换入

算法：
OPT 替换选择：将来最不可能用到的页；在最久的将来才用到的页-》未来事件-〉OPT不可实现-》衡量其他算法的标准-〉用历史事件预测未来

LRU 最近最不常使用（局部性原理）-》将来最不常使用
页什么时候被用过的问题？-》（1）时间戳（2）访问栈-〉代价很大-》近似算法-〉CLOCK

FIFO
最简单算法
内存区域（进程）-》循环缓冲区-〉指针，下一次用于替换的帧

CLOCK（针对LRU的改进）
用一个使用位为U来近似模拟时间戳 U=0/1
优先选择U=0的页替换（最近没被使用）
寻找：第一个U=0的页（指针，指向当前替换帧的下一个位置，下一次开始寻找的开始位置）-》动作：在遇到U=1时，要把U=1改成U=0
***U的初始值为1（默认用时才加载，请求式读取）
替换的页需要写回到辅存，IO操作，耗时
-》最好是能替换U=0的，并且不需要回写的帧-〉改进CLOCK-》CLOCK+
min=1 max=n+1

CLOCK+
两个位：U/M
替换的优先顺序：
U M
0 0
0 1
1 0
1 1
寻找：
（1）U=0，M=0的帧
（2）找U=0，M=1的帧 动作：遇到U=1-》改为U=0
（3）没有找到，转（1）《-所有变成U=0
min=1 max=3n+1
